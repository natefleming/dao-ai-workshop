# Lab 1 -- Your First DAO-AI Agent

**Level:** L100

## Goals

- Read a 30-line `greeter.yaml` and understand each top-level section.
- Auto-derive a per-student `username` and inject it via `params={...}`.
- Compile the YAML to a runnable agent with `AppConfig.from_file(...).as_graph()`.
- Deploy as a Databricks App with one call: `config.deploy_agent(target=DeploymentTarget.APPS)`.

## Deliverable

A live `hardware-store-<your-username>` Databricks App that responds to `"Hi! Is this thing on?"`.

---

**Use case:** `hardware_store` -- a customer-facing greeter for ACME Hardware.

**DAO-AI concept:** Declarative single-agent LLM. You describe the agent in YAML; `AppConfig.from_file(...)` builds a runnable LangGraph workflow.

## What you'll learn

- The four top-level YAML blocks every DAO-AI config has: `parameters:`, `resources:`, `agents:`, `app:`.
- How `AppConfig.from_file(path, params={...})` substitutes `${var.NAME}` references at load time.
- How `username` (auto-derived from your Databricks short name) keeps your deployed app name unique in a shared workspace.
- The deploy loop: `config.deploy_agent(target=DeploymentTarget.APPS)` -- one call generates the Asset Bundle, deploys it, and runs the app.

## Files

| File | Purpose |
|---|---|
| `greeter.yaml` | The agent config -- one LLM, one agent, no tools. |
| `notebook.py` | Walk through the YAML, build the agent, test, deploy. |
| `databricks.yaml` | Asset Bundle generated by `config.deploy_agent(...)`. |

## Prerequisites

- `databricks-claude-sonnet-4-5` foundation-model endpoint enabled.
- A Databricks Apps-eligible workspace (Serverless v5).
- No catalog or external resources required.

## Run

Open `notebook.py` on Serverless compute and run cell by cell. The notebook auto-derives `username` from your Databricks short name, builds the agent, invokes it locally, then deploys.

Deployed app name: `hardware-store-<your-username>` (e.g. `hardware-store-jane-doe`).
